# IELTS Speaking Data Contracts

Use these data contracts when a command reads or updates IELTS Speaking data. This file defines shared data meanings only; command steps live in `references/workflows/`.

## Data Directory

Default to the current workspace `speakingData/` directory.

- `speaking_profile.json`: user background, preferences, constraints, and reusable personal facts.
- `speaking_question_bank.json`: current-season IELTS Speaking questions.
- `speaking_answers.json`: approved saved answers only.
- `speaking_story_bank.json`: reusable personal stories, mainly for Part 2.
- `speaking_phrase_bank.json`: natural spoken expressions.
- `speaking_opinion_bank.json`: Part 3 idea modules.
- `speaking_practice_records.json`: practice attempts and feedback.
- `speaking_review.json`: review schedule for saved questions.

## Target Band

Path: `speaking_profile.json` -> `profile.speaking_preferences.target_band`

Meaning: the IELTS Speaking band target used for answer language level and feedback expectations.

Default: `7.0-7.5` when missing or empty.

Update rule: only `/ielts-speaking level [target band]` should modify this field.

## Question Status

Use only these question statuses:

- `new`: no approved answer has been saved.
- `saved`: an approved answer exists but it is not fluent yet.
- `practiced`: the user has spoken it at least once, but it is unstable.
- `fluent`: the user can say it smoothly but should still review it.
- `exam_ready`: the answer is ready for exam use and only needs light checks.

## Drafts And Saved Answers

Drafts generated by `/ielts-speaking answer`, `/ielts-speaking add`, or `/ielts-speaking edit` are editable conversation drafts.

Write to `speaking_answers.json` only after the user runs `/ielts-speaking save` or clearly asks to save the current approved answer.

## Review Intervals

Use these spaced-review intervals for saved materials:

- `review_count` 0 -> next review +1 day
- `review_count` 1 -> next review +3 days
- `review_count` 2 -> next review +7 days
- `review_count` 3 -> next review +14 days
- `review_count` 4 -> next review +30 days
- `review_count` 5 -> mastered

If marked forgot, reset `review_count` to 0 and set `next_review` to tomorrow.

## Part 2 Story Bank

Only extract or update stories after a Part 2 answer has been approved and saved.

Do not save a story from an early draft or from an answer the user is still revising.
